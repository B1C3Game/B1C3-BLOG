B1C3-BLOG Design Document
Purpose: Scalable blog scaffold for B1C3 founder content. Ship HTML + CSS + Markdown-to-HTML pipeline. No Hugo, no build complexity.

Repository: B1C3Game/B1C3-BLOG
Deploy target: https://b1c3game.github.io/B1C3-BLOG/
Current live B1C3 home: https://b1c3game.github.io/multi-agent-collab-test/
DNS note: b1c3.dev is not the live target yet and should not be treated as current production navigation.

Design Principles
Minimal but complete — HTML + CSS only. No frameworks. No build step (until needed).
Design consistency — Shared header/nav/footer language with the current live landing page at https://b1c3game.github.io/multi-agent-collab-test/. Same typography, colors, spacing, and atmospheric treatment.
Content-first — Posts live as .html files or simple markdown. Easy to add new posts without touching code.
Scalable structure — Directory layout supports unlimited posts. Add new post = add one file + update index.
No external dependencies — GitHub Pages serves static files. No server-side rendering, no backend.
Implementation note: the current repo has moved from the early generic scaffold to a landing-page-aligned visual system. Some example blocks further down are historical notes and should be treated as rough reference, not source of truth.

4. README.md (For Humans & Copilot)
Instructions:

To add a new blog post:

Create a new file: posts/[post-slug].html
Copy structure from posts/manifesto.html
Replace content section with your post
Add an entry to index.html posts list:
Copy
<article class="post-card">
  <h2><a href="posts/[post-slug].html">Your Post Title</a></h2>
  <p class="date">Published: YYYY-MM-DD</p>
  <p class="excerpt">Brief summary...</p>
  <a href="posts/[post-slug].html" class="read-more">Read →</a>
</article>
Push to GitHub. Deploy automatic (GitHub Pages).
To modify design:

Edit css/style.css to change shared styles
Edit css/responsive.css for mobile/tablet breakpoints
Test locally in browser (just open index.html)
Deployment: GitHub Pages auto-deploys on push to main. Target site URL is https://b1c3game.github.io/B1C3-BLOG/

Scalability Plan
Phase 1 (Ship Now):

Manifesto as first post
2-3 navigation links
Clean, minimal design
Manual post addition (edit HTML files)
Phase 2 (When Needed):

Add Python script to convert markdown → HTML (no build tool, just a helper)
Automatic index generation (script reads posts/ directory, updates index.html)
Archive/category system (if 20+ posts)
Phase 3 (If Content Explodes):

Migrate to static site generator (Hugo, Jekyll, 11ty) only if >50 posts
But keep the same design/structure — no UX change
Design Consistency Checklist
Header/nav matches b1c3.dev exactly
Color palette pulled from landing page CSS
Typography (font family, sizes) shared
Footer design consistent
Link colors + hover states match
Spacing/padding scales match
Mobile responsive (matches landing page breakpoints)
Success Criteria
Manifesto published target: https://b1c3game.github.io/B1C3-BLOG/posts/manifesto.html
Blog homepage lists all posts with excerpts
Link from b1c3.dev to blog works cleanly
Adding new post takes <5 minutes (just HTML file + one entry)
Design feels like continuation of b1c3.dev, not separate app
